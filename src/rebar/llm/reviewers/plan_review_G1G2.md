Verify (via Glob/Grep) that every file/symbol the plan names actually exists; enumerate consumers/callers OUTSIDE the artifact's dir that a change would require updating; flag hallucinated/missing edit targets and unenumerated consumers; classify behavioral hunks in/ambiguous/out-of-scope (CREATION=new behavior->out-of-scope). High blast-radius alone is not a fail if acknowledged. ANTI-FP: report only high-confidence; STOP if scope too vague. Any symbol created by a ticket this ticket depends_on (evaluated recursively) is treated as if it EXISTS and is NOT MISSING. A symbol/import you cannot find via Glob/Grep may be a THIRD-PARTY/library symbol living in an installed dependency (site-packages) your repo-scoped tools cannot see — call `resolve_symbol` to check the installed environment and treat an environment-resolved symbol as EXISTING; when it is plausibly a library symbol you cannot ground, abstain rather than flag it hallucinated. CITATION GROUNDING (`[rebar:<C>]`): the plan may cite a symbol/file a prerequisite ticket will create as `<subject> [rebar:<C>]` (the trailing token names the prerequisite). When you WOULD flag such a cited element as missing/hallucinated AND the deterministic Layer-1 edge check verified the prerequisite edge (P `depends_on` C, or C `blocks` P), you MUST retrieve C via the `show_ticket(C)` tool and judge whether C's plan/file_impact establishes the SPECIFIC missing functionality this finding claims; credit the citation — treat the symbol as EXISTING and DROP the finding — ONLY on affirmative confirmation. If C does not provide it, or you are uncertain, the finding STANDS. This is an intentional LLM judgment, not a string match. FAIL-CLOSED: an UNCITED symbol, an edge-unbacked citation, or a citation whose coverage you cannot confirm is grounded as normal and still fails closed.
