## Findings (Critical â†’ Low)

### [Critical] MCP `run_workflow` bypasses `REBAR_MCP_ALLOW_LLM` for v3 branch/loop/map LLM steps  (bug) â€” EMPIRICALLY CONFIRMED
- **Citations:** `src/rebar/llm/workflow/runs.py:285-308` (`has_llm_steps._scan`), `src/rebar/mcp_server.py:946` (the gate), `src/rebar/schemas/workflow.v3.schema.json` (`branch.then/else`, `loop`/`map` bodies are bare step **arrays**), `src/rebar/llm/reviewers/plan_review_finder.md:8` (`execution_mode: single_turn`), `src/rebar/llm/runner.py:198-211` (single_turn path skips `assert_gated`).
- **Failure scenario:** write-enabled MCP server, `REBAR_MCP_ALLOW_LLM` unset. `_scan` recurses into a construct only via the `{steps:[...]}` dict shape; a v3 `then: [<step>...]` is a *list* value hit by `if not isinstance(s, dict): continue`, so LLM steps inside branch arms are invisible. Reproduced: `has_llm_steps` returns **False** for both `gates/plan-review.yaml` and `gates/completion-verification.yaml`. The fence at `mcp_server.py:946` is skipped; plan-review's single-turn finders (up to ~31 criteria + verifier + coach) then make live billable calls with no gate.
- **Nuance (from verification):** *agentic* nested steps fail closed at `assert_gated` (so `completion-verification.yaml` errors before its billable call), but plan-review's `single_turn` steps set `tools=[]` and never reach that fence. The documented billing/policy gate is bypassed on a default-shipped workflow â€” Critical stands.
- **Tradeoffs:** the detector predates the v3 bare-array schema; fixing `_scan` to treat `then`/`else`/`body` arrays as frames is small but must not miss nesting.
- **Recommendation:** in `_scan`, recurse into the list values of `then`/`else`/`body` directly; add a v3-shape regression test (the only existing coverage, `test_agent_op_safeguard.py:93`, exercises the legacy v2 `then:{steps:[...]}` shape only). **Fix through Gerrit with a test â€” not a janitor edit.**

### [High] `docs/llm-framework.md` describes the superseded child-closure gate semantics  (docs)
- **Citations:** `docs/llm-framework.md:381-395` vs `src/rebar/llm/completion.py:56-130` (`_child_closure_findings` returns `(blocking, uncertified)`), `src/rebar/llm/workflow/gate_ops.py:44-100` (precheck emits `certifiable=False`, LLM still runs), `src/rebar/_commands/transition.py:279-285` (PASS + `certifiable is False` â†’ close **without** signing), `src/rebar/schemas/completion_verdict.schema.json:37`.
- **Justification:** The doc says any child not closed+certified â‡’ "FAIL verdict immediately, without ever invoking the LLM" and "the LLM evaluator is only ever reached once all children are closed+**signed**." HEAD: only an **unclosed** child is a deterministic FAIL; a closed-but-uncertified (force-closed) child now lets the parent close *unsigned* with `certifiable=False`, and the LLM runs on the parent's own criteria. The workflow YAML's own description (`gates/completion-verification.yaml:5-6`) contradicts the ops it wires. An agent following the doc will expect `--force-close` where none is needed and misread verdicts.
- **Recommendation:** Rewrite `llm-framework.md:381-395` around the blocking-vs-certification-withholding split; document the `certifiable` field. (Not a mechanical one-liner â€” needs accurate new semantics; left for a reviewed doc change.)

### [High] Module-size CI gate has no growth ratchet for allowlisted files  (size, temporal)
- **Citations:** `.github/workflows/test.yml:106-133` (gate); `.github/module-size-allowlist.txt` (6 entries). Growth since 2026-06-25 (direct baseline diff): `llm/plan_review/attest.py` 268â†’821 (**+553**), `config.py` 1048â†’1310 (+262), `reconcile.py` +173, `mcp_server.py` +101.
- **Justification (verification settled a cross-agent contradiction):** the gate re-checks **all** `src/rebar/**.py` every run and fails any non-allowlisted file >800 â€” so an existing file that grows past 800 *does* fail. The one true blind spot is the allowlist itself: once a file is on it, there is **no** growth check. In 8 days the allowlisted set absorbed ~1,100 net LOC; `attest.py` tripled and was allowlisted the moment it crossed. `reconcile.py` is now ~2Ă— the cap.
- **Recommendation:** Change the allowlist format to `path max_loc`, seed ceilings at current sizes, and fail the gate when an allowlisted file exceeds its recorded ceiling (a small % tolerance preserves maintenance headroom).
