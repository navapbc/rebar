# rebar plan-review gate — full criteria registry (authoritative, v4 §5)

A plan-review gate that fires when a ticket is claimed (open->in_progress), reviewing the ticket's IMPLEMENTATION PLAN before an agent executes it. Coaching/advisory posture. Four execution tiers: DET (deterministic code), 1-TURN, 2-STEP, AGENT (tool-using, inspects the live codebase). Below is the complete set.

## Layer 1 — DET (deterministic, blocks, fail-open) — run as code, not LLM
- P1 readiness-shape (standardized `## Acceptance Criteria`, all types)
- P2 file/symbol/import resolution (LSP/ast-grep)
- P3 package existence (syft + registry + LLM-fallback)
- P4 oversize signals (AC/file counts, conjunctions, layers/interfaces; scc/lizard)
- P5 task-DAG validity + cross-task interference (shared file/symbol/table/config-key)
- P6 AC/DD quality (compound-AND split, vague lexicon, spec-fidelity, verify-command robustness)
- P7 destructive/irreversible sniff (escalates to T4)

## Cross-cutting
- COH coherence pass (cross-section contradictions; single structured check)
- BROAD bounded open-ended pass (unknown-unknowns; advisory, one pass)
- benign-reading FP filter + anti-fabrication (dispositions in the system prompt)

## Posture notes
- ADVISORY/coaching (not roadblocks); only the DET tier blocks. Per-criterion PASS/AMBIGUOUS/FAIL.
- Findings authored to a feedback spec: grounded in a criterion/location, specific+actionable, severity-ranked+capped, affirm what passes, suggested fix only when confident, checklist-form.
- Proportionate scrutiny by ticket LEVEL (epic/story/task) + TYPE (bugs exempt; test-tasks suppress E5; internal-leaf suppress F4) via an applies_at field.
- Binary-checklist decomposition where sub-questions decompose cleanly; graded/narrative only where genuinely qualitative.
